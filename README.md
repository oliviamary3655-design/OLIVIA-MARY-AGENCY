Olivia Mary Digital Agency - Portfolio
This repository contains the source code for the Olivia Mary Digital Agency portfolio website. The site is designed to showcase premium web development, digital strategy, and research services.
Overview
This website provides a modern, high-performance, and responsive digital presence for the agency. It features:
Hero Section: High-impact introduction highlighting the agency's value proposition.
Core Services: Detailed breakdown of our web development, e-commerce, and research capabilities.
Portfolio (Works): A filterable showcase of past projects across multiple industries (Church, Legal, Healthcare, etc.).
Staff Profiles: Professional introductions for the agency founders, Olivia Mary and Caleb Kehinde.
Lead Generation: A structured contact form to streamline incoming project requests.
Technical Details
The portfolio is built with a "Performance-First" philosophy, focusing on:
Clean CSS Grid & Flexbox: Ensures a responsive layout across all device sizes.
Accessibility & Speed: Optimized for quick loading times without compromising on aesthetic quality.
CSS-only Animations: Utilizes keyframes for smooth transitions, hover states, and orb background effects, ensuring compatibility without the need for heavy JavaScript libraries.
How to Deploy
Due to the structure of the code, this site is designed to be hosted on platforms that support modern web standards.
1. GitHub Pages / Netlify / Vercel (Recommended)
These platforms are ideal for this static HTML/CSS structure.
Upload the `index.html` file to your repository.
Connect your repository to your chosen hosting service.
The site will be live instantly with full functionality.
2. WordPress.com (Note on Restrictions)
If you are using the Free Plan on WordPress.com, note that the platform heavily restricts custom code.
The provided code is "WP-Safe", meaning it removes JavaScript and restricted tags that WordPress.com typically blocks.
To use it: Paste the code into a "Custom HTML" block.
Limitation: You may notice that certain advanced visual effects (like specific CSS filters) might be sanitized or altered by the WordPress platform's built-in CSS security policies.
